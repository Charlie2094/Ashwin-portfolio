📋 Data Flow — Step by Step
1User Input — User fills form in Power Apps Canvas App and submits a request.
2Flow Triggered — Power Automate flow activates on form submission.
3Memory Fetch — Flow queries SharePoint list to retrieve previous email chain and conversation history.
4Prompt Construction — Chain-of-thought prompt is assembled: system role + email history + current request + reasoning steps.
5AI Inference — Structured prompt is sent to LLaMA 3.8B Instant — model reasons step-by-step using CoT.
6Response Parsed — AI output is extracted and formatted for email.
7Email Sent — Power Automate sends the AI-generated email via the configured email connector.
8Memory Updated — Response saved back to SharePoint to maintain conversation continuity.
9UI Updated — Power Apps displays the result to the user — loop complete.

📋 Agent Execution Flow
1Task Input — User enters a task via Streamlit web UI.
2GitHub Orchestration — Code versioned on GitHub, GitHub Actions handles CI/CD deployment pipeline.
3Orchestrator Activated — Agent Orchestrator receives task, breaks it into a multi-step plan.
4ReAct Loop Begins — Agent enters Reason → Act → Observe cycle using the ReAct pattern.
5Tool Selection — Orchestrator selects appropriate tool per step (search, code, file, API, memory).
6Observation — Tool result is fed back to the agent as context for the next reasoning step.
7Loop or Complete — Agent loops until task is complete or max steps reached.
8Final Output — Result rendered in Streamlit UI and logged to session memory.
